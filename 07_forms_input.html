Formulários e Inputs

Os formulários são elemento do HTML que nos permitem pedir dados ao usuário para, então, podermos validar e processar. Para que os formulários funcionem são necessárias 3 instâncias: Capturar (HTML), Validar (JS + Back-end), Processar (Back-end)

Os atributos de "form" são "action" e "method". ACTION define a rota em que será processada a informação capturada. METHOD define como a informação será enviada. Na versão atual do HTML, há apenas dois valores possíveis: GET e POST Observações sobre GET: Anexa os dados do formulário à URL, em pares nome/valor. NUNCA use GET para enviar dados confidenciais! (os dados do formulário enviado são visíveis na URL!). O comprimento de um URL é limitado (2048 caracteres). Útil para envios de formulários em que um usuário deseja marcar o resultado GET é bom para dados não seguros, como strings de consulta no Google. Observações sobre o POST: Anexa os dados do formulário dentro do corpo da solicitação HTTP (os dados do formulário enviado não são mostrados na URL). O POST não tem limitações de tamanho e pode ser usado para enviar grandes quantidades de dados. Envios de formulários com POST não podem ser marcados. Dica: Sempre use POST se os dados do formulário contiverem informações confidenciais ou pessoais!


O elemento INPUT é multifuncional. Alterando o valor do atributo type, podemos obter diferentes tipos de campos. O atributo name identifica o campo e o diferencia dos demais campos. É importante saber que os elemento que não têm name não serão enviados. A largura padrão do campo de input é de 20 caracteres.




O elemento label serve para associar um texto descritivo a um determinado campo. * FOR associa o label com um campo através da sua propriedade id. * CONTEÚDO contém o texto que será exibido e indica as informações que o usuário deve preencher no campo. * ID DO CAMPO serve para que o label seja associado, o campo que queremos associar deve possuir um id com o mesmo valor usado no for * LABEL, em geral, cada campo ou grupo terá seu próprio label associado. Quando o usuário clicar no label, será ativado automaticamente o campo ao qual ele corresponde; ajudando aqueles que tem dificuldade em clicar em regiões pequenas. O label também é muito útil para os usuários de leitores de tela, porque o leitor irá leer em voz alta o valor contido no label quando o usuário focar no elemento input.

Data

Dentro dos atributos do elemento, a data sempre é formatada como aaaa-mm-dd, por exemplo, 2022-03-08. Por fora, ela terá o formato que corresponde ao idioma do ususário. Também é possível definir uma data minima "min" que pode ser escolhida. Ou um data máxima "máxima".





Type Radio

Representa UMA opção selecionavel com um clique. Para criar o tipo radio, usamos o input cujo atributo type tenha valor radio. É possível agrupar os elementos radio, mas o susário só poderá escolher uma opção dentre eles. Para agrupar, é só declarar o mesmo valor para o atributo name em todos os elementos. Para que uma opção apareça pré-selecionada é só usar o atributo "checked"

The Submit Button

O input type submit define um botão para submeter o formulário para um manipulador de formulário, o form-handler. O form-handler é normalmente um arquivo no servidor com um script para processamento de input de dados. Ele é especificado no atributo "action" do form.






O atributo TARGET

O atributo target especifica onde será exibida a resposta recebida após o envio do formulário. Ele pode ter um dos seguintes valores: "_blank" = A resposta é mostrada em uma nova janela oui aba. "_self" = A resposta é exibida na janela atual. "_parent" = A resposta é exibida no frame principal. "_top" = A resposta é exibida em todo o corpo da janela. "framename" = A resposta é exibida em um quadro nomeado.






Grouping Form Data with Fieldset

O fieldset elemento é usado para agrupar dados relacionados em um formulário. O legend elemento define uma legenda para o fieldset.

The datalist Element

Especifica uma lista de opções predefinidas para um elemento input elemento. Os usuários verão uma lista suspensa das opções predefinidas à medida que inserem dados. O atributo "list" do "input" deve se referir ao id do "datalist".

Preenchimento automático

Preenchimento automático - O autocompleteatributo especifica se um formulário deve ter o preenchimento automático ativado ou desativado. Quando o preenchimento automático está ativado, o navegador completa automaticamente os valores com base nos valores que o usuário inseriu anteriormente. -- "form action="/action_page.php" autocomplete="on"" --

Atributo Novalidate - O novalidateatributo é um atributo booleano. Quando presente, especifica que os dados do formulário (entrada) não devem ser validados quando enviados. -- form action="/action_page.php" novalidate --
